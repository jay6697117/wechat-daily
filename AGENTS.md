# wechat-daily 项目级 Codex 指令

> [!CAUTION]
> 本文件适用于当前目录及其所有子目录。所有项目沟通、说明、总结默认使用简体中文；代码、命令、路径、文件名和标识符保持原样。

## 项目目标

本仓库用于按微信群和日期整理聊天记录，并基于聊天内容生成日更漫画、派生图片和对应的 `index.html` 页面。

## 日更漫画目录结构

- 第一层按群名称分类，例如：`AI全书学习交流群/`。
- 第二层按日期 `YYYY-MM-DD` 分类，例如：`AI全书学习交流群/2026-05-25/`。
- 日期目录内保存：
  - 当天聊天记录。
  - 剧本 Markdown。
  - 漫画 PNG。
  - 页面入口 `index.html`。

## 漫画生成流程

1. 先用 `wechat-cli` 导出当天完整聊天记录。
2. 基于完整聊天记录生成剧本。
3. 用户确认剧本后，再生成漫画图和 `index.html`。
4. 漫画页数不能固定，必须由当天聊天量和剧本实际分镜决定。
5. 每张漫画图必须严格按对应剧本页的 `第 N 格` 内容生成，不要自由改剧情。
6. 批量生成前，优先先生成并核查第一页。
7. 第一页确认后，再继续生成后续页面。

## 图片生成设置

- Codex 优先使用项目内 skill：`.codex/skills/codex-gateway-imagegen`。
- helper 脚本路径：`.codex/skills/codex-gateway-imagegen/scripts/generate_gateway_image.py`。
- 漫画页尺寸固定为 `1024x1536`。
- 生成方式优先使用参考图编辑：`--action edit`。
- 参考图优先使用 `AI全书学习交流群/2026-05-24/` 下的漫画图，尤其适合参考对白框和角色舞台的页面。

## 视觉风格基准

后续 `AI全书学习交流群` 日更漫画必须参考 `AI全书学习交流群/2026-05-24/` 的漫画图风格：

- 竖版漫画页。
- 顶部白底黑边标题条，黑色粗体标题。
- 底部白底黑边吐槽条，两侧可有小星星装饰。
- 中间按剧本实际分镜数量纵向排列，不固定三格。
- 每个分镜使用粗黑边框。
- 深蓝宇宙、数据夜空、群聊舞台背景。
- 手机群聊界面只能作为局部叙事道具，不能做整页真实微信截图。
- 左侧或边缘可放圆形群友头像、头像列表、消息卡片、警告牌、速度线和夸张表情。
- 对白气泡必须像参考图：白色圆角气泡、黑色描边、气泡尾巴指向说话者。
- 对白气泡旁边要有彩色昵称牌。
- 每格至少保留 1-2 个来自剧本的关键对白；长对白可以压缩，但不能换说话人或改剧情。

## 固定角色设定

生成每一页时都必须带上固定角色设定，防止角色串形象。

- `沙雕鱼`：永远是灰银色大嘴鱼吉祥物，圆眼、张大嘴、喜剧表情。昵称牌只能写“沙雕鱼”。任何其他角色的对白气泡尾巴都不能指向鱼。
- `凌封`：黑色冷静头像、黑色小人、技术判官气质。昵称牌写“凌封”。不能画成鱼。
- `张小灰`：群聊侦探、漫画记录者，戴放大镜或拿分镜板、Codex 检测仪的小头像。昵称牌写“张小灰”。不能画成鱼。
- `kio`：刚睡醒的账号盲盒管理员，困倦小头像，带盲盒、仓库钥匙等道具。昵称牌写“kio”。不能画成鱼。
- `流光 / Garen / 破防组`：普通圆形群友头像，可被账号墓碑压住或抱键盘。昵称牌按剧本写对应名字。不能画成鱼。
- `叭啦叭啦`：后端开发小人，搭配代码屏、IoT 线缆、小说稿纸、乱码窗口等道具。昵称牌写“叭啦叭啦”。不能画成鱼。
- `Sailor`：踩坑幸存者，搭配小公司天坑地图、小说书架等道具。昵称牌写“Sailor”。不能画成鱼。
- `Syngnat / 重启宇宙 / kkkk / yyh / 蔚蓝 / LX / 其他发言人`：统一画成小圆形技术弹幕头像，保留对应昵称牌。不能画成鱼。
- `Claude / Codex / Agent / AI 工具`：机器人、终端、检测仪或软件图标形象。不能画成鱼。

## 角色一致性硬规则

- 鱼只代表 `沙雕鱼`。
- 如果画面里出现鱼，要么给它“沙雕鱼”昵称牌，要么让它作为无对白吉祥物出现。
- 其他角色的对白气泡不能指向鱼。
- 每个对白气泡的尾巴必须指向对应说话者。
- 剧本里的角色、对白和画面元素优先级高于生成模型自由发挥。
- 发现角色串形象时，从第一页开始核查并重生，不要在错误基础上继续生成。

## 内容安全和表现限制

- 账号封禁、验证、接码、账号买卖、token、网络节点、公益站等内容只能表现为群聊背景笑点。
- 不写真实 URL、手机号、交易价格、平台链接、token 字符串、批量注册、绕过步骤或具体教程。
- 不画真实群友肖像。
- 不做整页真实微信截图。
- 手机界面、消息气泡、系统警报牌只能作为漫画叙事道具。

## `index.html` 生成和验证

- `index.html` 参考前一天同群目录下的页面风格。
- 更新标题、导航、图片列表、页码和 footer 目录。
- 必须验证：
  - 日期目录里所有预期 PNG 都存在且非空。
  - `index.html` 引用的图片路径都存在。
  - 本地浏览器加载页面无 console error/warn。
  - 每张漫画图片请求返回 `200 image/png`。

## Codex 执行约定

- 对已有聊天记录、剧本、图片和 HTML 做修改前，先确认当前文件状态，避免覆盖用户刚生成的产物。
- 生成或替换图片时，优先写入临时文件；尺寸、存在性和内容方向确认后，再替换稳定文件名。
- 修改非平凡流程时，同步记录关键发现、操作进度和验证结果到项目规划文件。
- 若用户最新明确指令与本文件流程有差异，可以按用户指令调整执行顺序；但不得突破角色一致性、内容安全和可验证性要求。
